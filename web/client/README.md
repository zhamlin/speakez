# Notes
## Firefox
When `privacy.resistFingerprinting` is enabled, there might be issues
with getting the sample rates to be correct from the audio context and the media stream.

Either disable `privacy.resistFingerprinting` or add the domain to `privacy.resistFingerprinting.exemptedDomains`
